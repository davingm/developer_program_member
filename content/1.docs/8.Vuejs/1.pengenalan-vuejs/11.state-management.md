# Modul 11: State Management (Pinia)

**Pinia** adalah store resmi untuk Vue.js. Ia memungkinkan Anda menyimpan data yang dapat diakses oleh komponen mana pun di aplikasi Anda tanpa harus mengirimkan props secara manual.

## Mengapa Butuh Pinia?

Bayangkan Anda memiliki data pengguna yang sedang login. Data ini dibutuhkan di:
- Header (untuk menampilkan nama).
- Profile Page (untuk edit data).
- Checkout (untuk alamat pengiriman).

Tanpa Pinia, Anda harus mengirimkan data ini lewat banyak level komponen (*prop drilling*). Dengan Pinia, semua komponen mengambil dari satu sumber yang sama.

---

## Membuat Store Pertama

Biasanya disimpan di folder `src/stores/`.

```js [src/stores/counter.js]
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useCounterStore = defineStore('counter', () => {
  // State
  const count = ref(0)

  // Getters (seperti computed)
  const doubleCount = computed(() => count.value * 2)

  // Actions (seperti methods)
  function increment() {
    count.value++
  }

  return { count, doubleCount, increment }
})
```

---

## Menggunakan Store di Komponen

Sangat mudah dan bersih:

```vue
<script setup>
import { useCounterStore } from '@/stores/counter'

const counter = useCounterStore()
</script>

<template>
  <div>
    <p>Hitungan: {{ counter.count }}</p>
    <p>Dua Kali Lipat: {{ counter.doubleCount }}</p>
    <UButton @click="counter.increment">Tambah</UButton>
  </div>
</template>
```

## Tips Penggunaan

::tip{icon="i-lucide-shield-check"}
Jangan pernah mengubah state langsung dari komponen jika logikanya kompleks. Selalu buat **Action** di dalam store agar logika bisnis Anda terpusat dan mudah diuji.
